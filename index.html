Welcome to Anipy’s docs!¶

Anipy is a python library that wraps and organize the Anilist rest api into modules, classes and functions so it can be used quick, easy, and right out of the box. You can take a look at the api official docs. Anilist is a Josh Star’s project

class anipy.core.Resource[source]¶

Abstract resource class.

Works as a base class for all other resources, keeping the generic and re-usable functionality.

Provides to the classes that inherit it with a connection pool (urllib3.connectionpool.HTTPSConnectionPool) and methods to make all requests to the anilist api through it.

_ENDPOINT = None¶

Default endpoint for each resource implementation.

_URL = 'http://anilist.co'¶

Constant. Base url that is used for all requests. Each resource must define it own endpoint based on this url.

_headers¶

Generates the default headers with the according credentials.

Example:

{
    "Authorization": "BEARER youraccestokenhere",
    "Content-Type": "application/json"
}
Returns:(dict) Default headers for common requests.

request(method, endpoint=None, data=None, headers=None)[source]¶

Makes a method request to endpoint with data and headers.

Parameters:
  • method – (str) String for the http method: GET, POST, PUT DELETE. Other methods are not supported.
  • endpoint –

    (str, optional) String for the endpoint where the request aims. Remember, all endpoints refers to _URL.

    If none, request will aim to _ENDPOINT.

    Example: ‘/api/user/demo/animelist/’

  • data –

    (dict, optional) Parameters to be included in the request.

    If none, no parameters will be sent.

  • headers –

    (dict, optional) Headers to be included in the request.

    If none, default parameters will be used (see _headers).
Raise:

See raise_from_response
Returns:

(dict) Response.
